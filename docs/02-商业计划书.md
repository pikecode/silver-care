## 一、项目概述

### 1.1 一句话定位

> 基于微信小程序的社区居家养老健康管理平台，通过"子女远程关怀 + 老人日常打卡 + 社区驿站服务"三位一体模式，解决 3 亿居家老人的健康管理和子女远程关怀需求。

### 1.2 核心价值主张

| 用户 | 痛点 | 家邻康的解法 |
|------|------|-------------|
| **老人** | 吃药易忘、健康无人管、求助不方便 | 定时提醒、一键打卡、一键呼叫 |
| **子女** | 不在身边、不了解父母状况、担心又帮不上 | 每日健康报告、异常预警、远程代约服务 |
| **社区** | 养老服务数字化程度低、管理靠纸笔 | 居民健康档案、活动管理、服务工单系统 |

### 1.3 项目阶段

当前处于：**产品设计 → MVP 开发阶段**

## 二、市场分析

### 2.1 目标市场规模（TAM → SAM → SOM）

```
TAM（潜在市场）
  中国银发经济总量：7 万亿（2024）→ 30 万亿（2035）

SAM（可服务市场）
  社区居家养老健康管理：约 3000 亿
  （3 亿老人 × 健康管理年均消费 1000 元）

SOM（可获取市场）— 3 年目标
  100 个社区 × 平均 2000 户 × 10% 渗透率 × 年均 350 元
  = 约 700 万元/年
```

### 2.2 目标用户画像

#### 主要付费者：子女（25-45 岁）

- 一二线城市上班族，与父母不同城或忙于工作
- 月收入 8000-30000 元
- 愿意为父母健康支出 30-100 元/月
- 高频使用微信，习惯线上支付

#### 核心使用者：老人（55-80 岁）

- 有 1-3 种慢性病（高血压、糖尿病居多）
- 使用微信（主要聊天、看文章、发红包）
- 不愿安装新 App，但接受小程序
- 社区活跃度中等，有基本社交需求

### 2.3 政策环境

| 政策 | 要点 | 对我们的影响 |
|------|------|-------------|
| 国务院银发经济意见（2024） | 首次提出"银发经济"概念，鼓励社会资本参与 | 政策红利期，利好 |
| "十四五"养老规划 | 每个社区配建养老服务设施 | 社区驿站有政策基础 |
| 老年人数字化专项 | 推进适老化改造 | 微信小程序符合方向 |
| 医保个人账户改革 | 个人账户可用于家庭成员 | 未来可对接医保支付 |

### 3.2 MVP 功能清单（v1.0）

**Phase 1 — 核心功能（第 1-4 周）**

| 功能 | 父母端 | 子女端 | 优先级 |
|------|--------|--------|--------|
| 用药提醒与打卡 | 收到提醒 → 点"已服药" | 查看打卡记录 | P0 |
| 血压/血糖记录 | 手动输入或语音输入 | 查看趋势图表 | P0 |
| 健康日报 | — | 每晚自动推送 | P0 |
| 异常预警 | — | 漏打卡/数据异常时推送 | P0 |
| 家庭绑定 | 被邀请加入 | 扫码绑定父母 | P0 |

**Phase 2 — 社区功能（第 5-6 周）**

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 社区活动列表 | 展示附近驿站活动 | P1 |
| 活动报名 | 父母端一键报名 | P1 |
| 一键呼叫子女 | 大按钮，直接拨打电话 | P1 |

**Phase 3 — 服务功能（第 7-8 周）**

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 上门服务预约 | 子女代约/老人自约 | P2 |
| 服务评价 | 完成后评分评价 | P2 |
| 会员订阅 | 付费解锁高级功能 | P2 |

### 3.3 适老化设计原则

```
1. 字体：最小 18px，关键信息 24px+
2. 按钮：最小 48x48px 触摸区域，间距 12px+
3. 颜色：高对比度，避免纯灰色文字
4. 操作：核心流程不超过 3 步
5. 语音：支持语音输入血压/血糖数值
6. 反馈：每次操作都有明确的视觉 + 震动反馈
7. 导航：底部固定 Tab，不超过 4 个入口
```

## 四、商业模式

### 4.1 收入模型

```
┌──────────────────────────────────────────────────┐
│                  收入来源                          │
├──────────┬────────────┬────────────┬─────────────┤
│ C端订阅   │ 服务佣金    │ 政府采购    │ 增值合作     │

│ 购买会员   │ 每单抽佣    │ 信息化建设  │ 精准推荐     │
│          │ 10-15%     │ 项目制     │ 按效果付费   │
│ ¥29/月   │            │            │             │
│ ¥299/年  │            │            │             │
└──────────┴────────────┴────────────┴─────────────┘
```

### 4.2 定价策略

| 套餐 | 价格 | 包含内容 |
|------|------|---------|
| **基础版（免费）** | ¥0 | 用药提醒（2 种药）、基础打卡、1 条/天异常通知 |
| **关怀版（主推）** | ¥29/月 或 ¥299/年 | 无限用药提醒、详细健康报告（日/周/月）、多人关注、优先客服 |
| **尊享版** | ¥59/月 或 ¥599/年 | 关怀版全部 + 每月 2 次上门量血压 + 专属健康顾问 |

### 4.3 单位经济模型（Unit Economics）

```
以单个付费家庭（关怀版年付 ¥299）计算：

收入
  年订阅收入：¥299
  上门服务佣金（预估 2 单/月 × ¥10）：¥240
  年均收入/家庭：约 ¥540

成本
  获客成本（CAC）：约 ¥50（线下驿站获客，成本极低）
  服务成本：约 ¥120/年（短信/通知 + 云服务摊销）
  运营人力分摊：约 ¥100/年

毛利/家庭：约 ¥270/年
毛利率：约 50%

LTV（假设平均使用 3 年）：¥810
LTV/CAC = 16.2（远超健康值 3）
```

## 五、运营策略

### 5.1 获客策略（Growth）

```
第一阶段：单社区深耕（0-6 个月）

  线下
  ├── 社区驿站合作（免费提供数字化工具换取场地）
  ├── 免费健康检测活动（血压/血糖）
  ├── 健康讲座（邀请社区医生）
  └── 社区公告栏海报

  线上
  ├── 社区微信群推广（通过驿站工作人员）
  ├── 子女朋友圈分享（"今日妈妈按时服药啦"）
  └── 微信公众号健康科普内容

第二阶段：社区复制（6-18 个月）

  ├── 标杆社区数据包装 → 说服更多社区合作
  ├── 老人口碑推荐（最有效的渠道）
  └── 对接街道/居委会批量导入
```

### 5.2 留存策略（Retention）

| 策略 | 机制 | 目标 |
|------|------|------|
| 每日打卡提醒 | 微信服务通知，固定时间 | 日活打卡率 > 60% |
| 打卡连续激励 | 连续打卡 7 天/30 天获得称号 | 7 日留存 > 50% |
| 子女周报 | 每周日推送本周健康总结 | 子女周活跃 > 40% |
| 社区活动 | 定期线下活动引导回流 | 月活跃 > 70% |
| 健康知识推送 | 每日一条适老健康贴士 | 内容打开率 > 30% |

### 5.3 社区驿站合作模式

```
家邻康提供：
  ✓ 免费数字化管理工具（居民档案、活动管理）
  ✓ 健康活动策划支持
  ✓ 物料（海报、易拉宝、宣传单）
  ✓ 服务商对接

驿站提供：
  ✓ 场地（活动、咨询、展示）
  ✓ 工作人员协助推广
  ✓ 居民信任背书
  ✓ 日常运营协助

分成：上门服务佣金驿站分 30%
```

## 九、风险分析

### 9.1 风险矩阵

| 风险 | 可能性 | 影响 | 应对策略 |
|------|--------|------|---------|
| **老人使用率低** | 中 | 高 | 极简设计 + 线下培训 + 子女代操作 |
| **社区合作难** | 中 | 高 | 提供免费数字化工具换取合作 + 对接居委会 |
| **付费转化低** | 中 | 中 | 先做免费口碑 → 自然转化；子女端营造"孝心"场景 |
| **大厂入局** | 低 | 高 | 大厂做不了线下社区运营，这是我们的壁垒 |
| **政策变化** | 低 | 中 | 当前政策利好，保持合规 |
| **服务质量失控** | 中 | 高 | 严格服务商准入 + 用户评价体系 + 保险兜底 |
| **数据安全** | 低 | 高 | 健康数据加密存储 + 最小权限原则 + 合规审查 |

### 9.2 最大风险与对策

**风险：老人不愿意/不会用**

这是所有银发产品的核心挑战。我们的三层解法：

```
第一层：产品极简
  → 核心操作 3 步以内，大字大按钮

第二层：子女帮设
  → 子女扫码帮父母开通，父母只需要"点一下"

第三层：线下兜底
  → 驿站工作人员手把手教，每周定时到驿站做免费检测
  → 即使老人不用小程序，也能通过驿站录入数据
```
